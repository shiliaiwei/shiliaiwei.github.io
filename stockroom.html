Database Design & Relationships

Core Tables Structure

Users Table

users
├── id (PK)
├── username (UNIQUE)
├── email (UNIQUE)  
├── password (HASHED)
├── full_name
├── role (ADMIN/MANAGER/EMPLOYEE)
├── status (ACTIVE/INACTIVE)
├── created_at
└── updated_at

Categories Table

categories
├── id (PK)
├── name (UNIQUE)
├── description
├── status (ACTIVE/INACTIVE)
├── created_at
└── created_by (FK → users.id)

Suppliers Table

suppliers
├── id (PK)
├── name
├── contact_person
├── email
├── phone
├── address
├── status (ACTIVE/INACTIVE)
├── created_at
└── created_by (FK → users.id)

Products Table

products
├── id (PK)
├── sku (UNIQUE)
├── name
├── description
├── category_id (FK → categories.id)
├── supplier_id (FK → suppliers.id)
├── unit_price
├── minimum_stock_level
├── current_stock_quantity
├── status (ACTIVE/DISCONTINUED)
├── created_at
├── updated_at
└── created_by (FK → users.id)

Stock Transactions Table

stock_transactions
├── id (PK)
├── transaction_type (IN/OUT)
├── product_id (FK → products.id)
├── quantity
├── unit_price
├── total_amount
├── reference_number
├── notes
├── transaction_date
├── created_by (FK → users.id)
└── created_at

Database Relationships

One-to-Many Relationships:

  • Users → Categories (1:N)
  • Users → Products (1:N)
  • Users → Stock Transactions (1:N)
  • Categories → Products (1:N)
  • Suppliers → Products (1:N)
  • Products → Stock Transactions (1:N)

Key Constraints:

  • Unique constraints on SKU, username, email
  • Foreign key constraints with referential integrity
  • Check constraints for positive quantities and prices

Project Success Criteria

This plan provides a solid foundation to build a production-ready inventory management system. Start with the backend database and entities, then build the APIs, and finally create the React frontend to consume those APIs.

MVP Success Criteria

Must Have (MVP)

  • User can login/logout
  • Admin can manage products and categories
  • Users can record stock in/out transactions
  • System shows current stock levels
  • Basic dashboard with key metrics

Nice to Have (Future Versions)

  • Advanced reporting & analytics
  • Mobile responsive design
  • Email notifications for low stock
  • Stock forecasting
  • Barcode scanning
  • Export reports to PDF/Excel
