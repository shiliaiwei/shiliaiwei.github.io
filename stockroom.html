StockRoom - Complete MVP Plan

Project Overview: Building a StockRoom Inventory Management System from scratch using Java Spring Boot (Backend) and React (Frontend).

MVP Core Features

Essential Features (Phase 1)

User Authentication & Authorization

  • Login/Register system
  • Role-based access (Admin, Manager, Employee)

Product Management

  • Add/Edit/Delete products
  • Product categories
  • SKU generation
  • Basic product information

Stock Management

  • Stock in (receiving inventory)
  • Stock out (issuing inventory)
  • Current stock levels
  • Low stock alerts

Basic Reporting

  • Stock levels report
  • Recent transactions
  • Simple dashboard

System Architecture

Frontend (React)     ←→     Backend (Spring Boot)     ←→     Database (MySQL)
    ├── Auth Pages            ├── REST APIs                  ├── Users
    ├── Dashboard             ├── Security Layer             ├── Products  
    ├── Product Pages         ├── Business Logic             ├── Categories
    ├── Stock Pages           ├── Data Access Layer          ├── Transactions
    └── Reports               └── Validation                 └── Suppliers

Database Design & Relationships

Core Tables Structure

Users Table

users
├── id (PK)
├── username (UNIQUE)
├── email (UNIQUE)  
├── password (HASHED)
├── full_name
├── role (ADMIN/MANAGER/EMPLOYEE)
├── status (ACTIVE/INACTIVE)
├── created_at
└── updated_at

Categories Table

categories
├── id (PK)
├── name (UNIQUE)
├── description
├── status (ACTIVE/INACTIVE)
├── created_at
└── created_by (FK → users.id)

Suppliers Table

suppliers
├── id (PK)
├── name
├── contact_person
├── email
├── phone
├── address
├── status (ACTIVE/INACTIVE)
├── created_at
└── created_by (FK → users.id)

Products Table

products
├── id (PK)
├── sku (UNIQUE)
├── name
├── description
├── category_id (FK → categories.id)
├── supplier_id (FK → suppliers.id)
├── unit_price
├── minimum_stock_level
├── current_stock_quantity
├── status (ACTIVE/DISCONTINUED)
├── created_at
├── updated_at
└── created_by (FK → users.id)

Stock Transactions Table

stock_transactions
├── id (PK)
├── transaction_type (IN/OUT)
├── product_id (FK → products.id)
├── quantity
├── unit_price
├── total_amount
├── reference_number
├── notes
├── transaction_date
├── created_by (FK → users.id)
└── created_at

Database Relationships

One-to-Many Relationships:

  • Users → Categories (1:N)
  • Users → Products (1:N)
  • Users → Stock Transactions (1:N)
  • Categories → Products (1:N)
  • Suppliers → Products (1:N)
  • Products → Stock Transactions (1:N)

Key Constraints:

  • Unique constraints on SKU, username, email
  • Foreign key constraints with referential integrity
  • Check constraints for positive quantities and prices

🎨 UI Design Principles

  • Clean Layout: Consistent spacing, clear typography, and intuitive navigation
  • Color Coding: Green for positive actions/stock in, Red for negative actions/alerts, Blue for neutral actions
  • Responsive Design: Mobile-first approach with flexible grid layouts
  • User Experience: Minimal clicks to complete tasks, clear visual feedback, and error handling
  • Accessibility: High contrast colors, keyboard navigation, and screen reader friendly
