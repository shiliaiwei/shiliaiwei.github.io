Game Theory

You're Always Playing, Whether You Notice or Not

Every human interaction is a game. Not with joysticks — with choices, consequences, and hidden moves. Whether you're negotiating salary, deciding whether to message first, or choosing a seat at the table, you're making strategic decisions based on what others might do. Life is one big multiplayer strategy game disguised as "normal life."

Your Best Move Isn't About You — It's About Others

In Game Theory, your best decision depends on what others are likely to pick. If everyone guesses on the counter, you choose the uncommon. If competitors lower prices, you either follow or create a premium product. Success isn't about what you want — it's about predicting what others will do and responding smarter.

People Follow Incentives, Not Morality

Game Theory proves people act based on payoffs. Not what's fair. Not what's right. What rewards them best. That's why people ghost, bargain, betray or help — depending on what benefits them most in that situation. Change the reward, and you change the behavior.

Zero-Sum Thinking Shrinks The Pie

Not every game is I win, you lose. Most situations grow value when people collaborate or make different moves. Yet Game Theory people often wrongly treat situations as zero-sum — and end up hurting themselves and everyone else. Great players look for positive-sum games where everybody eats.

When Games Repeat, Everything Changes

One-time decisions tempt risky moves. But in repeated games — like business, relationships, or reputation — betrayal backfires. People remember patterns. Smart players build trust early, punish betrayal, and focus on long-term gains over short-term wins. The second game always matters more than the first.

Your Gut Is Often Wrong

Game Theory teaches that what feels right often isn't. Playing hardball, hoarding advantages, or moving first might seem smart, but the best move often involves counterintuitive strategies: delay action, appear weak, or help competitors. Great players win by thinking several moves ahead, not by instinct.

Network Cheatsheet

Essential networking reference guide for developers and system administrators

Networking Fundamentals

OSI Model Layers

  • Layer 1: Physical - Cables, hubs, repeaters
  • Layer 2: Data Link - Switches, MAC addresses
  • Layer 3: Network - Routers, IP addresses
  • Layer 4: Transport - TCP/UDP
  • Layer 5: Session - NetBIOS, PPTP
  • Layer 6: Presentation - SSL/TLS, encryption
  • Layer 7: Application - HTTP, FTP, SMTP

Network Types

  • LAN: Local Area Network
  • WAN: Wide Area Network
  • MAN: Metropolitan Area Network
  • PAN: Personal Area Network
  • WLAN: Wireless Local Area Network

Routing Protocols

Interior Gateway Protocols (IGP)

  • RIP v1/v2: Distance Vector, 15 hop limit, metric = hop count
  • EIGRP: Cisco proprietary, hybrid protocol, fast convergence
  • OSPF: Link State, hierarchical design, uses LSAs
  • IS-IS: Link State, used in ISP networks, OSI protocol
  • IGRP: Cisco proprietary, legacy distance vector

Exterior Gateway Protocols (EGP)

  • BGP v4: Path Vector, Internet backbone, AS-to-AS routing
  • eBGP: External BGP, between different autonomous systems
  • iBGP: Internal BGP, within same autonomous system
  • EGP: Legacy exterior protocol (obsolete)

Routing Concepts & Metrics

  • Static Routing: Manually configured routes, no automatic updates
  • Dynamic Routing: Automatically learned routes, adaptive
  • Default Route: 0.0.0.0/0 - Route of last resort
  • Administrative Distance (AD): Route trustworthiness (0-255)
  • Metric: Cost calculation for best path selection
  • Convergence: Time for all routers to agree on topology
  • Load Balancing: Equal/unequal cost path distribution

Switching Protocols & Technologies

Spanning Tree Protocols

  • STP (802.1D): Original spanning tree, 50s convergence
  • RSTP (802.1w): Rapid spanning tree, 6s convergence
  • MSTP (802.1s): Multiple spanning tree, per-VLAN
  • PVST+: Cisco per-VLAN spanning tree plus
  • Rapid PVST+: Cisco rapid per-VLAN STP

VLAN Protocols & Standards

  • 802.1Q: VLAN tagging standard, 4-byte header
  • ISL: Cisco Inter-Switch Link (legacy), 30-byte header
  • VTP: VLAN Trunking Protocol, VLAN database sync
  • DTP: Dynamic Trunking Protocol, auto trunk negotiation
  • Native VLAN: Untagged VLAN on trunk (default VLAN 1)
  • VLAN Range: 1-4094 (1 & 4095 reserved)

Link Aggregation Protocols

  • LACP (802.3ad): Link Aggregation Control Protocol
  • PAgP: Cisco Port Aggregation Protocol
  • EtherChannel: Cisco link bundling technology
  • Static LAG: Manual link aggregation configuration
  • Load Balancing: SRC-MAC, DST-MAC, SRC-DST-IP, etc.

Switching Features & Protocols

  • CDP: Cisco Discovery Protocol, Layer 2 neighbor discovery
  • LLDP: Link Layer Discovery Protocol (IEEE standard)
  • BPDU Guard: Protects against STP loops on access ports
  • Root Guard: Prevents inferior BPDU reception
  • PortFast: Immediate forwarding for access ports
  • UDLD: UniDirectional Link Detection
  • Storm Control: Broadcast/multicast traffic limiting

Firewall & Security Protocols

Firewall Types & Technologies

  • Packet Filter: Layer 3/4 inspection, stateless filtering
  • Stateful Inspection: Connection state tracking, dynamic rules
  • Application Gateway: Layer 7 proxy, deep packet inspection
  • NAT Firewall: Network Address Translation with filtering
  • NGFW: Next Generation Firewall, integrated security
  • WAF: Web Application Firewall, HTTP/HTTPS protection
  • UTM: Unified Threat Management, all-in-one security

Security Protocols & Standards

  • IPSec: IP Security suite, ESP/AH protocols
  • SSL/TLS: Secure Socket Layer/Transport Layer Security
  • WPA2/WPA3: WiFi Protected Access (AES encryption)
  • WEP: Wired Equivalent Privacy (deprecated, RC4)
  • 802.1X: Port-based network access control (EAP)
  • RADIUS: Remote Authentication Dial-In User Service
  • TACACS+: Terminal Access Controller Access Control System
  • Kerberos: Network authentication protocol (ticket-based)

Intrusion Detection & Prevention

  • NIDS: Network Intrusion Detection System
  • NIPS: Network Intrusion Prevention System
  • HIDS: Host-based Intrusion Detection System
  • HIPS: Host-based Intrusion Prevention System
  • Snort: Open source IDS/IPS engine
  • Suricata: High performance IDS/IPS/NSM engine
  • SIEM: Security Information Event Management

Access Control & Authentication

  • ACL: Access Control Lists (standard/extended)
  • RBAC: Role-Based Access Control
  • MAC: Mandatory Access Control
  • DAC: Discretionary Access Control
  • LDAP: Lightweight Directory Access Protocol
  • Active Directory: Microsoft domain services
  • SSO: Single Sign-On authentication

Subnetting Quick Reference

  • Subnetting Formula: 2^n = subnets (n = borrowed bits)
  • Host Formula: 2^h - 2 = hosts (h = host bits)
  • Network ID: First address in subnet
  • Broadcast ID: Last address in subnet
  • Block Size: 256 - subnet mask octet value
  • VLSM: Variable Length Subnet Masking
  • FLSM: Fixed Length Subnet Masking

VPN Types & Technologies

VPN by Deployment Type

  • Site-to-Site VPN: Connects entire networks securely over internet
  • Remote Access VPN: Individual users connect to corporate network
  • Client-to-Site VPN: Software client connects to VPN server
  • Host-to-Host VPN: Direct secure connection between two devices
  • Extranet VPN: Secure access for business partners
  • Intranet VPN: Secure internal network connections

VPN Protocols & Technologies

  • IPSec VPN: Layer 3 encryption, ESP/AH protocols, IKE key exchange
  • SSL/TLS VPN: Web-based, clientless access, HTTPS encryption
  • PPTP: Point-to-Point Tunneling (legacy, insecure, port 1723)
  • L2TP/IPSec: Layer 2 tunneling with IPSec encryption (port 1701)
  • OpenVPN: Open-source SSL/TLS based VPN (UDP 1194)
  • WireGuard: Modern, lightweight VPN protocol (UDP)
  • SSTP: Secure Socket Tunneling Protocol (Microsoft)
  • IKEv2: Internet Key Exchange version 2 (mobile-friendly)

Cloud VPN Services & Solutions

  • AWS VPN: Site-to-Site & Client VPN, VPC connectivity
  • Azure VPN Gateway: Point-to-Site & Site-to-Site, ExpressRoute
  • Google Cloud VPN: Classic & HA VPN, Cloud Interconnect
  • Cisco AnyConnect: SSL VPN client, endpoint security
  • Fortinet FortiClient: SSL VPN solution with security fabric
  • Palo Alto GlobalProtect: Next-gen VPN with threat prevention
  • SonicWall NetExtender: SSL VPN thin client
  • Check Point VPN: Enterprise VPN with threat prevention

How VPN Works (Step-by-Step)

  1. Tunnel Establishment: Secure encrypted tunnel created between endpoints
  2. Authentication: User/device credentials verified (PSK, certificates, etc.)
  3. Key Exchange: Cryptographic keys negotiated (IKE, ISAKMP)
  4. Encapsulation: Original packets wrapped in VPN headers
  5. Encryption: Data encrypted using agreed algorithms (AES, etc.)
  6. Transmission: Encrypted packets sent through public network
  7. Decryption: Data decrypted at destination endpoint
  8. Delivery: Original packets delivered to final destination

VPN Security & Encryption

  • AES: Advanced Encryption Standard (128, 192, 256-bit)
  • 3DES: Triple Data Encryption Standard (legacy)
  • RSA: Public key cryptography for key exchange
  • SHA: Secure Hash Algorithm for data integrity
  • MD5: Message Digest 5 (deprecated, weak)
  • DH: Diffie-Hellman key exchange groups
  • Perfect Forward Secrecy: Unique keys per session
  • Split Tunneling: Selective traffic routing through VPN
